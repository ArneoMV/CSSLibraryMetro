Images

It is unnecessary to display large images on screens designed for smaller images. This issue has been resolved in the HTML5 standard by using two new attributes within the tag: srcset and sizes. The srcset attribute allows the tag to point the browser to multiple files. Sizes determines the width of the screen and the image that will be used for that screen width. This technique saves device memory and displays better quality images.


CSS explanation
<img> - tag uses four attributes: srcset, sizes, src, and alt
<figure> & <figcaption> - Inside the HTML, we used a <figure> tag that served to add an <figcaption> element that allowed us to print the image or media title
<src> - is used to find the root of the image we are showing
<alt> - is used to print the name of an image if the image cannot be displayed on the screen
<srcset> - attribute serves to retrieve a set of images for different screen images within

<figure> <img srcset="../img/slika320.png 320w, ../img/slika480.png 480w, ../img/slika720.png 800w" sizes="(max-width: 320px) 280px, (max-width: 480px) 440px, 800px" src="../img/slika320.png" alt="slika"> <figcaption>slika 320wpx</figcaption> </figure> HTML
