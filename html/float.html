Float

The float CSS property places an element on the left or right side of its container, allowing text and inline elements to wrap around it

To understand its purpose and origin, we can look to print design. In a print layout, images may be set into the page such that text wraps around them as needed. This is commonly and appropriately called "text wrap".

ClearFix

Micro clearfix generates pseudo-elements and displays them on a table. This creates an anonymous table-cell and a new block formatting context, which means that before the pseudo-element prevents the collapse of the upper margin. A pseudo-element is used to clear floats. As a result, there is no need to hide the generated content and the total amount of code required is reduced.

  • Today's markup experts use this technique to avoid clutter in the markup. This technique was invented by Nicolas Gallagher. Read his original post on the micro-clearfix hack
  • In this technique, the parent element is given some magical CSS properties to clean the floats of all its children.
  • Flexbox is supported by Firefox 18, Chrome 21, Opera 12.10, and Internet Explorer 10, Safari 6.1 (including Mobile Safari) and Android's default browser 4.4.

Including the: before themenu, it's not necessary to delete floats, but it prevents the upper margins from collapsing in modern browsers.

    This has two advantages:

  • It provides visual consistency with other float retention techniques that create a new block formatting context, overflow: hidden;
  • Provides visual consistency with IE 6/7 when the zoom is applied to 1

Clear

  • Deleting floats by adding a clear element
  • Adding a clear element after a float (s) is the most common way people use to delete floats in CSS and you might already implement that thing in your markup. In this technique, we basically use a clear element to clean the floats of siblings. The clear element is usually used after floating siblings with no baby elements or content in it.
