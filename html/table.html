Table

A table is a summary of information organized into rows and columns. They are used to store and display data in a structured format. For example, databases store data inside a table so that data can be accessed quickly. Websites use tables to display multiple rows of data. Spreadsheets combine both tables to manipulate data and display it in a structured format.

When creating a web spreadsheet, it should be answered what the user is trying to display with the spreadsheet. We've found that the user has three common goals when interacting with a spreadsheet:

  • The user wants to search as much information as possible as quickly as possible
  • The user wants to perform actions quickly, such as deleting rows, downloading items
  • Data comparison

It is possible for the table to be filled with a large amount of data that will become opaque on a mobile device. To solve this problem, we used the static horizontal left-side table header technique.

The following tables change the orientation on small screens. Horizontal scrolling or dragging is used to display more data.Table head is always visible.

The technique for responsive design is to use display: { inline-block; } on row tables and white-space: { nowrap; } on the tbody element.

Brakepoint for tablet screens where the table is being transformed is 840px
