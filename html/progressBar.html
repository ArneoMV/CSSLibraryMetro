Progress Bar

The progress bar is a graphical component that is used to visualize the progress steps when executing an action. The bar is accompanied by a textual presentation of progress in percentage format. The purpose of the progress bar is to show the user where they are during their journey through the execution of the action. It plays an important role in reducing the anxiety that many users feel when performing their action.



In the paper, "The Importance of Percentage-Done Progress Indicators for Computer-Human Interfaces," Dr. Brad Myers of the University of Toronto found that most users wanted to have a status indicator. Research has shown that users are more pleased to see their progress. Progress indicator increases customer engagement. When respondents from the no-progress group were asked if they wanted any indication of progress, 75 percent said yes. So, users want to have a progress bar. The answer to the question of why progress bar is such a powerful element can be found in the human subconscious. People have goals, and they want to meet those goals. The user will feel pleased when something is achieved.

attribute explanation
<progress> element can be used in conjunction with JavaScript to display the progress of a task or process as it is in progress
<progress> usually written as <progress value = "0" max = "100"> </progress> with any content within the start and end tags. This content may be the current value and / or maximum value that can be displayed to legacy browsers (i.e., browsers that do not support the <progress> element)
value attribute represents the current value (i.e. where progress is currently on). If you omit this attribute, the progress bar becomes "indefinite" and will not show any progress (but most browsers will show an animated bar that will show the user to try ...)
<max> attribute represents the total (i.e. where the progress bar value will be when completed).
