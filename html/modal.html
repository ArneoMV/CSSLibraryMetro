Modal window

A modal window helps to attract user attention, encourages user participation and provides relevant and useful information. It is the type of window that is subordinate to the parent window. A modal window is a secondary window that stands above the parent window and assumes parental control until the action is completed. The user is denied the ability to perform actions until the modal window is closed, but the parent window is still visible. The most common options offered by the modal window are: Close, Cancel, or Office.

    Modal windows are often used in the following cases:

  • Drawing attention to important data
  • Blocking the rest of the application until the condition for continuing to work is satisfied
  • Warning that the actions of the current action are not reversible

Modal window design is important to facilitate user journey through the application. We display the modal window after appropriate user action.
If the modal window appears suddenly without user action, then there will be no context in which the window appears and will cause confusion for the user. In this case, the user is most likely to close the window without looking closely at the information the window displays. For example, closing your document without saving is a pre-qualified modal window event that asks you if you want to save your document.
The modal window should gain user confidence with its design. There should be responsive design to ensure good functionality across all screen widths.

The modal window of the Metro work frame consists of three parts: head, body and base; within which we put the related information. The header is where the title of the window will be displayed. Inside the body, text will be displayed, and a footer with buttons for the user's actions. In the upper right corner is the "X" button to close the window. Modal windows can display different information of different importance. For this purpose, we have created six different types of windows that differ in color.
