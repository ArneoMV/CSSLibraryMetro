Button's

Buttons are an important element of UI design. A button can be any control that allows the user to trigger events. Their primary role is user-system interaction. When creating a button, it is necessary to follow the guidelines in order for the button to perform its purpose in a quality way.

    The button should look like a button
  • The user should know immediately what is clickable and what is not. Any element that requires the user to work hard to decipher what a particular element is doing is wasting the user's time. When interacting with a button, the button should show visual markings that it is a button, these can be resized, resized, colored, shadowed, etc.
    • Put the button where users expect it
  • Buttons should be located where users expect them or where they can find them. If the user cannot find the button, they will not know that the button exists.
    • The title of the button should describe it's operation
  • Users need to clearly understand what happens when they click a button.
    • The button should be of the correct dimensions
  • Button size indicates element priority. A large button indicates primary actions, while a smaller button indicates secondary actions. The primary button should be more prominent. We will use a contrasting color on the primary button to get the user's attention. The MIT Touch Lab study found that average toe pads are 10-14 mm in size and 8-10 mm in tip tips. Therefore, 10 mm x 10 mm is a good minimum target size for a finger to press.
    • Observe the order of interaction
  • Button ordering should follow user-system interaction
    • Avoid using too many buttons
    Provide visual interaction feedback
  • When a user clicks a button he expects the user interface to respond with feedback. If the user does not have feedback, they may realize that the system has not received their command and will retry.
  • When the user moves the cursor over the button, the user expects visual confirmation that the button can be pressed. This technique is possible on desktops, but not on mobile devices. Mobile devices can't hover over a button, we can just click it with our finger. For this purpose, for mobile devices the button should be made clear that it is a button to avoid confusion.

Six types of buttons

There are six typical types of buttons, each with four basic states of behavior: normal, focused, depressed, and disabled.

    1. Hover button
  • A characteristic of a hover button is its shadow. The shadow gives the impression of a dimension that indicates that it is possible to click a button. Use the hover buttons to highlight the button.
    2. Float button
  • The float button looks the same as the float button except that the float button has no shadow. Without shadows, disruptive content is reduced.
    3. Ghost button
  • The characteristic of the ghost button is its transparency. The body of the wrinkle spirit is translucent and is bounded by a line marking the edge of the button. When moving the cursor over the button, a transitional animation is created to fill the button with color.
    4. Text button
  • Text buttons reduce distractions. When you focus on a button, the text changes color.
    5. Icon button
  • The purpose of a button with an icon is to visually show what the button is for. The icon is placed in the middle of the circle.
    6. Toggle button
  • The Toggle button has two states. Depending on the condition, the appearance of the buttons will change.

Animations

The buttons should show visual feedback on the interaction. To do this, we created four different button animations that can be applied to each type of button.

Bounce
Shake
Grow
Shrink


class explanation
bounce shows the bounce animation
shake shows shake animation
grow shows a zoom animation
shrink shows a reduction animation
